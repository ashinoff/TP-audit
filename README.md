# Power Atlas — аудит привязок абонентов к ТП

Streamlit-приложение для поиска абонентов, которые с большой вероятностью
привязаны не к своей трансформаторной подстанции (ТП). Загружаете Excel-выгрузку —
получаете ранжированный список подозрительных привязок с обоснованием.

Файл обрабатывается **в памяти процесса и нигде не сохраняется**.

## Запуск локально

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Деплой на streamlit.app

1. Закоммитьте репозиторий (`app.py`, `requirements.txt`, `README.md`) на GitHub.
2. Зайдите на [share.streamlit.io](https://share.streamlit.io) → **New app**.
3. Укажите репозиторий, ветку и `app.py`.

## Входной файл

Приложение умеет читать **любые** выгрузки, не привязываясь к фиксированным именам
или порядку колонок. На стартовой странице показана схема: какие поля используются
и какие из них обязательны. Сопоставление выполняется автоматически по приоритетам:

| Приоритет | Логика                                                             |
|-----------|---------------------------------------------------------------------|
| 100       | Точное совпадение нормализованных строк (lowercase, ё→е, NFKC)      |
| 80        | Заголовок начинается с alias-префикса                               |
| 60        | Alias найден как отдельное слово в заголовке (`\b…\b`)              |
| 40        | Alias найден как подстрока в заголовке                              |

Дополнительно настроены `avoid_contains` — анти-синонимы, которые блокируют
неверные совпадения. Например, поле **ТП** никогда не схватит колонку с
**ТУ / ТУСТЕК**, даже если они стоят рядом.

После загрузки можно проверить и при необходимости вручную поправить сопоставление
прямо в интерфейсе.

### Какие колонки нужны

| Колонка             | Обязательно | Назначение                                  |
|---------------------|-------------|---------------------------------------------|
| ТП                  | да          | Идентификатор трансформаторной подстанции   |
| Подключение / ПС    | нет         | Питающая вышестоящая подстанция             |
| Фидер               | нет         | Линия 6/10 кВ от ПС до ТП                   |
| Населённый пункт    | нет         | Город, посёлок, село                        |
| Улица               | нет         | Название улицы или СНТ                      |
| Дом                 | нет         | Номер дома                                  |
| Лицевой счёт        | нет         | Для вывода в отчёт                          |
| Договор             | нет         | Для вывода в отчёт                          |
| № п/п, Код ТУ, ТУ   | нет         | Для вывода в отчёт                          |

Чем больше колонок передано — тем точнее анализ. **Без ТП работать не будет.**

## Алгоритм

Для каждого абонента вычисляется сумма баллов по пяти независимым сигналам.
Сигналы независимы — чем больше сработало, тем выше уверенность.

| Сигнал              | Балл  | Когда срабатывает                                                          |
|---------------------|-------|----------------------------------------------------------------------------|
| **ПС**              | 50    | Питающая ПС абонента ≠ доминирующей на его ТП                              |
| **Фидер**           | 50    | Фидер абонента ≠ доминирующего на его ТП                                   |
| **Нас. пункт**      | 20    | Нас. пункт ≠ доминирующего (если тот охватывает ≥ 80%)                     |
| **Улица-одиночка**  | 25    | Улица на ТП единственная, но имеет «домашнюю» ТП в этом же нас. пункте (≥ 5 абонентов) |
| **Адрес**           | 30    | Связка «улица + дом» в основном (≥ 75 %) относится к другой ТП (мин. 4 записи) |

Веса и пороги настраиваются в боковой панели.

### Интерпретация балла

- **≥ 100** — почти наверняка ошибка
- **50 – 99** — высокая вероятность, проверить вручную
- **30 – 49** — кандидат на проверку

## Что в выходном файле

Excel-отчёт с четырьмя листами:

1. **Подозрительные** — ранжированный список с баллами и причинами
2. **По ТП** — агрегация: где больше всего аномалий
3. **Статистика** — счётчики срабатываний сигналов
4. **Методика** — описание сигналов

## Что приложение **не** делает

- Не сохраняет данные на сервере — всё в памяти процесса
- Не отправляет данные в сторонние сервисы
- Не использует геокодинг — работает только с текстом из выгрузки

## Технологии

- **Streamlit** + кастомный CSS для оболочки
- **Pandas** + **OpenPyXL** для работы с Excel
- Шрифты: Fraunces (антиква), Instrument Sans (гротеск), JetBrains Mono (моно)
- Палитра: тёплый кремовый фон, угольный текст, медно-янтарный акцент
